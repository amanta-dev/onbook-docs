개요

viewer OnBook 뷰어는 교과서(ebook)·콘텐츠·내 수업하기를 제공하는 Windows 데스크톱 뷰어입니다. 같은 구성을 웹 서버에 올려 웹 버전으로도 사용할 수 있습니다.

이 문서는 콘텐츠 개발사가 OnBook 뷰어를 구성하고 활용하는 데 필요한 내용을 안내합니다.

이 문서에서 다루는 내용

  • OnBook 뷰어의 폴더 구조
  • 각 뷰어의 UI·부가 기능 설정 방법 — settings.json
  • 개발사 제작 페이지에서 뷰어 필수 라이브러리 (onbook.api.min.js)를 활용하는 방법 — 함수 상세는 API 가이드 참고
  • 패키징 산출물을 뷰어에 배치하는 방법 — 간단 안내, 자세한 절차는 패키저 가이드 참고
  • 설치본 구성·설정 방법

뷰어 폴더 구조

뷰어는 교과서 폴더 단위로 배포됩니다. 콘텐츠 개발사가 다루는 작업 영역은 다음과 같습니다. 

(교과서 폴더)/   ← 폴더명 규칙은 설치본 구성 참고
└─ platform/
   ├─ onbook/
   │  ├─ ebook/          ← 교과서 배치
   │  ├─ contents/       ← 콘텐츠 배치
   │  ├─ download/       ← 단원 다운로드 자료
   │  ├─ main/           ← 메인 화면 (개발사 제작·커스텀)
   │  ├─ modules/        ← 뷰어 모듈 (개발사 페이지 + 필수 모듈)
   │  └─ …
   ├─ scene/
   │  ├─ ebook/settings.json     ← ebook 뷰어 설정
   │  ├─ contents/settings.json  ← contents 뷰어 설정
   │  ├─ myclass/settings.json   ← myclass 뷰어 설정
   │  └─ … (뷰어 화면 앱 — 수정 금지)
   └─ … (실행 엔진 — 수정 금지)
초록으로 표시된 작업 영역 외의 파일·폴더(뷰어 화면 앱 내부, 실행 엔진 등)는 뷰어 동작에 필요한 구성 요소이므로 가급적 수정하지 말아 주세요.

책(권) 단위 폴더 규칙

ebook/과 contents/는 같은 책(권) 단위 폴더를 사용합니다 — unit01: 책 1, unit02: 책 2 (수학 익힘책처럼 2권으로 구성되는 경우).

  • 교과서: platform/onbook/ebook/unit01/
  • 콘텐츠: platform/onbook/contents/unit01/lesson01/ (책 폴더 아래 단원 폴더)

뷰어 주요 화면

제작한 교과서·콘텐츠가 뷰어에서 어떻게 표시되는지 화면별로 소개합니다.

메인 화면

주 메뉴 이동하는 화면
교과서 ebook 뷰어 — ebook 패키저로 만든 교과서가 표시됩니다.
콘텐츠 contents 뷰어 — contents 패키저로 만든 단원별 콘텐츠가 표시됩니다.
내 수업하기 myclass 뷰어 — 교과서·콘텐츠 페이지를 골라 수업을 구성하는 화면입니다.

ebook 뷰어 (교과서)

교과서 페이지를 넘기며 보는 화면입니다. 페이지 이동·썸네일 목록·그리기 도구를 제공합니다.

교과서 페이지에 배치한 버튼을 누르면 연결된 HTML 콘텐츠가 열립니다. 교과서(ebook 패키저)와 콘텐츠(HTML)가 만나는 지점이므로, 배치 후 버튼 연결이 정상 동작하는지 꼭 확인해 주세요.

contents 뷰어 (콘텐츠)

단원별 콘텐츠를 페이지 순서대로 보는 화면입니다. 페이지 순서와 썸네일은 contents 패키저에서 정한 목록을 그대로 따릅니다.

myclass 뷰어 (내 수업하기)

교과서·콘텐츠 페이지를 골라 나만의 수업을 구성하는 화면입니다. 별도 제작물 없이 배치된 교과서·콘텐츠를 재사용하며, 수업 목록은 뷰어 설정의 myclass.lessons로 정의합니다.

콘텐츠 개발 규칙

뷰어에서 정상 동작하는 HTML 콘텐츠를 만들기 위해 다음 규칙을 지켜 주세요.

onbook.api 로드

콘텐츠에서 뷰어 기능(페이지 이동 등)을 쓰려면 onbook.api.min.js를 로드합니다. 파일은 콘텐츠 폴더 안 common/js/ob/에 두고, 각 페이지에서 상대 경로로 로드합니다. 

<!-- 콘텐츠 HTML의 <head> 또는 <body> 하단 -->
<script src="../../common/js/ob/onbook.api.min.js"></script>
제공 함수 목록과 사용 방법은 API 가이드 — 사용 방법 · 전체 API 목록을 참고하세요.

페이지 이동 규칙

페이지 이동·링크는 onbook.api 함수(goPage 등)를 사용하고, window.location.href를 직접 변경하는 방식은 가급적 사용하지 말아 주세요. 뷰어의 페이지 이력·이동과 어긋날 수 있습니다.

콘텐츠나 외부 페이지에서 뷰어 화면으로 되돌아올 때는:

window.ob.viewer.api.historyBack();

화면·뷰포트 전제

뷰어는 화면 확대·축소(핀치 줌)를 사용하지 않습니다. 콘텐츠는 확대 없이 한 화면에 온전히 표시되도록 제작해 주세요.

페이지 파일명 관례

콘텐츠 페이지 파일명은 다음 형식을 따릅니다.

e{책}_{단원}_{차시}_{코너}_{탭}.html   예: e5_01_02_01_01.html

뷰어가 파일명에서 단원·차시 정보를 읽으므로, 이 관례를 지켜 주세요.

개발 체크리스트

  • common/js/ob/onbook.api.min.js를 각 페이지에서 상대 경로로 로드했다.
  • 페이지 이동·복귀에 onbook.api를 사용했다 (window.location.href 직접 변경 없음).
  • 확대 없이 한 화면에 온전히 표시된다.
  • 페이지 파일명이 명명 관례를 따른다.

뷰어 설정 (settings.json)

각 뷰어의 메뉴 구성·기능·모듈 연결은 뷰어별 settings.json으로 커스터마이징할 수 있습니다. 뷰어를 교과 특성에 맞게 조정하는 기본 파일입니다.

  • platform/scene/ebook/settings.json — ebook 뷰어
  • platform/scene/contents/settings.json — contents 뷰어
  • platform/scene/myclass/settings.json — myclass 뷰어

공통 설정 항목

항목 설명
viewerType 뷰어 종류. 수정하지 마세요.
menu.items 하단 메뉴에 표시할 항목과 순서. "---"는 구분선.
menu.features 이 뷰어가 지원하는 기능 전체 목록. items에 넣을 수 있는 항목의 참고 목록입니다.
quickMenu.items 퀵 메뉴바 구성 (ebook 뷰어).
rightMenu 오른쪽 메뉴의 위(top)·아래(bottom) 배치.
page 페이지 표시 — double(양면 여부), arrow(이동 화살표 표시·세로 위치) 등.
modules.main.url 홈(메인 화면) 경로.
modules.library 자료실 경로와 열기 방식(target·maximize).
modules.toolkit.items 활동 도우미 도구 목록. 도구별 label·icon·url·target·maximize를 지정해 추가·제거할 수 있습니다.
myData 내 자료함에 표시할 항목(메모·북마크·하이라이트 등).
print 인쇄 옵션 — silent(대화상자 없이 현재 페이지 인쇄), previewEnabled(미리보기).
draggableMenu / bottomMenu 메뉴 드래그 이동 허용, 하단 메뉴 사용 여부 (fixedSettingHelp: 설정·도움말을 항상 맨 끝에 고정).

myclass 전용 설정 — 수업 구성

myclass 뷰어의 수업 목록은 myclass.lessons에 정의합니다. 단원(lessons) → 차시(sessions) → 섹션(sections) → 코너(corners) 구조이고, 각 코너가 콘텐츠 페이지 하나에 연결됩니다. 

"myclass": {
  "lockContentLink": false,          ← 콘텐츠 페이지 이동 차단 여부
  "uiType": "top-only",            ← 메뉴 표시 (top-only | all)
  "lessons": [{
    "name": "Lesson 1",
    "title": "Lesson 1 Where Are You From?",
    "sessions": [{
      "name": "첫째 시간",
      "sections": [{
        "name": "도입",
        "corners": [{
          "title": "단원 열기",
          "pageCount": 8,       ← 교과서 쪽수 표시
          "contentUrl": "../../onbook/contents/unit01/lesson01&pageName=e5_01_01_01_01.html"
        }]
      }]
    }]
  }]
}

myclass에는 안내 문구를 바꾸는 messageCustom 항목도 있습니다.

settings.json은 JSON 문법(쉼표·따옴표)이 어긋나면 뷰어가 정상 동작하지 않습니다. 편집 후 뷰어를 다시 실행해 확인해 주세요.

메인 화면

platform/onbook/main/은 뷰어 실행 시 처음 표시되는 메인 화면으로, 콘텐츠 개발사가 직접 제작·커스터마이징 하는 영역입니다. 기본 제공 구성은 index.html + js/index.js(링크 설정) + css/·images/입니다.

기능 연결 방법

메인 화면에서 각 기능은 다음 방식으로 연결합니다. 기본 제공 js/index.js의 링크 설정(LINK_CONFIG)을 수정하면 대상 교과서·콘텐츠에 맞게 바꿀 수 있습니다.

기능 연결 방법
교과서 열기 (ebook) scene/ebook/index.html?contentUrl=(교과서 경로)&page=(쪽)
콘텐츠 열기 (contents) scene/contents/index.html?contentUrl=(단원 경로)&page=(순번)
내 수업하기 (myclass) scene/myclass/index.html?lesson=(단원)&session=(차시)
진도 보기 window.OPEN_PROGRESS_VIEW() 호출 — progress 모듈이 제공하는 함수입니다.
자료실 ../modules/library/index.html로 이동.
개발사 제작 HTML 새 창 window.ob.viewer.api.openUrl(url, 창이름, 옵션) — API 가이드 — openUrl 참고.
메인 화면 index.html이 로드하는 진도 보기(progress)·단원 다운로드(updateLesson) 모듈의 css/js 참조는 유지해 주세요. 제거하면 해당 기능이 동작하지 않습니다.

뷰어 모듈

platform/onbook/modules/는 메인 화면과 각 뷰어에서 사용하는 부가 페이지 모음입니다. 필수 모듈과 개발사 제작 영역으로 나뉩니다.

폴더 구분 설명
progress/ 필수 — 삭제 금지 진도 보기. 메인 화면이 직접 사용합니다.
updateLesson/ 필수 — 삭제 금지 단원 다운로드. 메인 화면이 직접 사용합니다.
library/ 개발사 제작 자료실 — 교과 자료를 모아 보여주는 페이지.
toolkit/ 개발사 제작 활동 도우미 — 칠판·타이머·모둠 점수판·발표 도우미 등 수업 도구 페이지.
apps/ 개발사 제작 기타 페이지 — 사전·게임·사용 설명서 등 자유 구성.
common/ 공용 모듈들이 함께 쓰는 리소스(스크립트·폰트·오디오 등).

모듈 연결 경로 규칙

  • 메인 화면에서: ../modules/… — 예: ../modules/library/index.html
  • 뷰어 설정(settings.json)에서: ../../onbook/modules/… — 예: ../../onbook/modules/toolkit/timer.html

패키징과 배치

패키징

완성한 제작물은 패키저로 ZIP을 만듭니다. 절차는 각 패키저 가이드를 참고하세요.

제작물 사용 도구
교과서 ebook 패키저 가이드
단원별 HTML 콘텐츠 contents 패키저 가이드

교과서(ebook) 배치

교과서 ZIP의 압축을 풀어 내용물이 책 폴더(unitXX/) 바로 아래에 오도록 배치합니다. 

platform/onbook/ebook/unit01/
├─ book.xml
├─ toc.xml
├─ book_tab/
└─ web/   ← 페이지 HTML · css · js · page_data · page_images · page_thumbnails

콘텐츠(contents) 배치

콘텐츠 ZIP의 압축을 풀어 내용물이 단원 폴더(lessonXX/) 바로 아래에 오도록 배치합니다. 자세한 내용은 contents 패키저 가이드 — 뷰어 배치를 참고하세요. 

platform/onbook/contents/unit01/lesson01/
├─ toc.xml
└─ web/   ← toc.xhtml · page_thumbnails · 원본 파일들

단원 다운로드 자료 (download/)

platform/onbook/download/는 뷰어의 단원 다운로드 기능이 사용하는 자료 폴더입니다. 구성 방법은 별도 안내를 따릅니다.

배치 후 확인

배치를 마치면 뷰어를 실행해 메인 화면에서 교과서 / 콘텐츠로 들어가 실제 표시를 확인합니다.

목록에 나타나지 않으면 압축 해제 시 폴더가 한 겹 더 생기지 않았는지 확인해 주세요. toc.xml이 unit01/(교과서) 또는 lesson01/(콘텐츠) 바로 아래에 있어야 합니다.

웹 버전으로 사용하기

PC에서 배치·확인을 마친 뷰어는 웹 서버에 올려 브라우저로도 사용할 수 있습니다. platform/ 안의 onbook/과 scene/ 두 폴더만 웹 서버에 나란히 업로드합니다. 

(웹 루트)/
├─ onbook/   ← 교과서·콘텐츠 포함
└─ scene/

브라우저 진입 주소는 다음과 같습니다.

https://(웹 루트)/onbook/main/index.html

접속하면 PC 버전과 동일한 메인 화면이 표시됩니다. 교과서·콘텐츠 표시를 같은 방법으로 확인하세요.

설치본 구성

USB 등 설치 매체로 배포할 때는 배포 루트에 설치 런처(Start.exe)와 설치 정보 파일이 함께 들어가며, 다음 항목을 개발사(출판사)가 정합니다.

설치 정보 (start.ini)

항목 설명
TITLE 교과서 이름. 배포 루트의 교과서 폴더명과 반드시 동일해야 합니다. 설치·실행 창 제목으로도 사용됩니다.
P_CODE 제품(출판사) 코드. 예: AMT1
SUBJECT_CODE 과목 코드 — {학교급}{과목}{학년}{저자구분} 형식. 예: EEN5A (초등·영어·5학년·A)
EDITION 교과서 판. 예: 2022개정
INSTALL_DIR 기본 설치 경로 (사용자가 설치 시 변경 가능). 예: C:\OnBook
REQUIRED_SPACE 설치에 필요한 최소 디스크 공간 (바이트).
DIRECT_RUN_MSG 설치하지 않고 직접 실행할 때 표시할 안내 문구.

설치 시 다음 경로로 복사됩니다.

{INSTALL_DIR}\{P_CODE}_{SUBJECT_CODE}_{EDITION}\{TITLE}\   예: C:\OnBook\AMT1_EEN5A_2022개정\초등학교 영어 지도서 5 전자책등\
start.ini와 autorun.inf는 반드시 EUC-KR 인코딩으로 저장해 주세요. UTF-8로 저장하면 한글이 깨집니다.

교체할 수 있는 요소

  • autorun.inf — label을 교과서명으로 지정.
  • images/ — 설치 화면 이미지(배경·버튼 비트맵). 버튼 위치는 start.ini의 [LAYOUT]으로 조정.
  • platform/boot/icon.png — 설치본 관리 목록에 표시되는 교과서 아이콘.
