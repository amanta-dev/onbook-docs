개요

단원 다운로드는 교과서 배포본에 모든 단원의 콘텐츠를 담는 대신, 필요한 단원만 골라 내려받아 설치하는 기능입니다. 배포본의 용량을 줄이고, 사용자는 수업에 필요한 단원부터 바로 사용할 수 있습니다.

  • 메인 화면에서 단원을 선택해 다운로드하면, 해당 단원의 콘텐츠가 뷰어에 설치됩니다.
  • 아직 다운로드하지 않은 단원의 콘텐츠는 열 수 없으며, 다운로드를 안내합니다.
  • 이 기능은 PC(설치형) 전용입니다. 웹 버전에서는 표시되지 않습니다.

이 문서는 단원 다운로드 테스트 담당자를 위해 기능의 동작 방식을 설명하고, 순서대로 따라 하며 확인할 수 있는 체크리스트형 시나리오를 제공합니다.

각 확인 항목의 체크 상태는 이 문서를 연 브라우저에 자동 저장됩니다. 창을 닫았다가 다시 열어도 체크한 내용이 유지되므로, 여러 번에 나누어 테스트해도 됩니다.

테스트 준비

테스트 배포본

단원 다운로드 테스트용 배포본은 다음과 같이 구성되어 있습니다.

  • 교과서 폴더 1개 — 실행 파일(Start.exe)과 뷰어가 포함되어 있습니다.
  • 단원 콘텐츠는 배포본에 포함되어 있지 않으며, 두 개 단원 (lesson01, lesson02)을 뷰어 안에서 다운로드해 설치합니다.

배포본 위치와 실행

  1. 배포본 위치를 엽니다:
    \\NAS2\0003_file_public\2025_Amanta Viewer\04_프로그램 관련\프로토개발\05_updatelesson\
  2. 안에 있는 교과서 폴더를 내 PC로 복사합니다. (NAS에서 바로 실행하지 마세요)
  3. 복사한 폴더 안의 Start.exe를 실행합니다.
다운로드한 단원은 뷰어가 있는 위치에 직접 설치됩니다. 따라서 USB에 담긴 뷰어를 USB에서 바로 실행하면 단원이 USB에 설치되어 설치 속도가 심하게 느려집니다. USB로 배포본을 받은 경우에도 반드시 PC에 복사한 후 실행해 주세요.

준비 사항

항목 내용
PC Windows PC. 배포본 폴더를 내 PC로 복사한 뒤 테스트합니다.
네트워크 단원 파일을 서버에서 내려받으므로 인터넷 연결이 필요합니다.
초기 상태 배포본을 처음 복사한 상태에서는 모든 단원이 미다운로드 상태입니다.

처음 상태로 되돌리기 (재테스트)

테스트를 처음부터 다시 하고 싶다면, 사용하던 배포본 폴더를 지우고 원본 배포본을 새로 복사하는 것이 가장 확실합니다.

배포본을 새로 복사하지 않고 되돌리려면: 뷰어를 종료한 뒤 배포본 폴더 안 platform\update-lesson.json 파일에서 해당 단원의 "downloaded": true를 false로 고치고, platform\onbook\contents\ 아래에 설치된 해당 단원 폴더를 삭제합니다. 익숙하지 않다면 배포본 재복사를 권장합니다.

동작 방식

단원 다운로드는 다음 순서로 동작합니다.

  1. 메인 화면의 단원 선택에서 단원을 고릅니다.
  2. 아직 다운로드하지 않은 단원이면 다운로드 버튼이, 이미 다운로드한 단원이면 다운 됨 표시가 나타납니다.
  3. 다운로드 버튼을 누르면 진행 창이 뜨고 다운로드 → 설치가 차례로 진행됩니다.
  4. 완료되면 "다운로드 완료!" 표시가 잠시 나타난 뒤 자동으로 닫히고, 해당 단원은 다운 됨으로 바뀝니다.
  5. 다운로드 상태는 뷰어를 껐다 켜도 그대로 유지됩니다.

미다운로드 단원의 접근 제한

아직 다운로드하지 않은 단원의 콘텐츠를 열려고 하면 아래 안내가 표시되고, 확인을 누르면 메인 화면으로 돌아갑니다.

"단원을 다운로드하지 않았습니다. 홈 화면에서 단원을 다운로드한 후 이용해 주세요."

다운로드 실패 시

네트워크 문제 등으로 다운로드에 실패하면 "단원 다운로드에 실패했습니다." 알림이 표시됩니다. 이 경우 네트워크 상태를 확인한 뒤 같은 단원을 다시 다운로드하면 됩니다.

화면 안내

메인 화면 — 단원 다운로드 영역

구성 요소 설명
단원 선택 다운로드할 단원을 고르는 목록입니다. 테스트 배포본에는 lesson01, lesson02 두 단원이 있습니다.
다운로드 버튼 선택한 단원이 미다운로드 상태일 때만 표시됩니다.
다운 됨 선택한 단원이 이미 다운로드된 경우 버튼 대신 표시됩니다.
메인 화면은 콘텐츠 개발사가 자유롭게 제작하는 영역이므로, 실제 제품에서는 단원 다운로드 UI의 디자인·위치가 다를 수 있습니다. 이 문서는 테스트 배포본의 화면을 기준으로 설명합니다.

진행 창 (다운로드 진행률)

다운로드 버튼을 누르면 화면 가운데에 진행 창이 뜨고, 아래 순서로 바뀝니다.

단계 표시 내용
① 다운로드 중... 단원 파일을 서버에서 내려받습니다. 진행률(%)과 진행 막대가 표시됩니다.
② 설치 중... 내려받은 파일을 뷰어에 설치합니다. 진행률(%)과 진행 막대가 표시됩니다.
③ 다운로드 완료! 완료 표시가 약 1.5초 나타난 뒤 자동으로 닫힙니다.

시나리오 1. 기능 노출 확인

단원 다운로드 UI가 올바른 환경에서만 보이는지 확인합니다.

PC(설치형)에서는 보여야 하고, 웹 버전에서는 보이지 않아야 합니다.

절차
  1. 배포본 폴더를 내 PC로 복사하고 Start.exe를 실행합니다.
  2. 메인 화면에서 단원 다운로드 영역(단원 선택 목록)을 찾습니다.
확인 항목

시나리오 2. 미다운로드 단원 접근 제한

다운로드하지 않은 단원의 콘텐츠를 열 수 없는지 확인합니다.

아무 단원도 다운로드하지 않은 초기 상태에서, 안내 메시지가 뜨고 메인 화면으로 돌아가야 합니다.

절차
  1. 아무 단원도 다운로드하지 않은 상태에서, 메인 화면 > 콘텐츠로 콘텐츠를 열어 봅니다.
확인 항목
2단원(lesson02)의 접근 제한은 콘텐츠 뷰어 안에서 이동해야 확인할 수 있으므로, 1단원을 다운로드한 뒤 시나리오 4에서 확인합니다.

시나리오 3. 단원 다운로드 (기본 흐름)

단원 하나를 다운로드하고 완료까지의 흐름을 확인합니다.

다운로드 → 설치 → 완료 → '다운 됨' 표시까지 자연스럽게 이어져야 합니다.

절차
  1. 단원 선택에서 lesson01을 선택합니다.
  2. 다운로드 버튼을 누릅니다.
  3. 진행 창이 닫힐 때까지 기다립니다.
확인 항목

시나리오 4. 다운로드한 단원 이용

다운로드한 단원의 콘텐츠가 정상 동작하는지 확인합니다.

설치된 콘텐츠가 빠짐없이, 깨짐 없이 열려야 합니다.

절차
  1. 시나리오 3에서 다운로드한 단원(lesson01)의 콘텐츠를 엽니다. (메인 화면 > 콘텐츠)
  2. 콘텐츠 뷰어 안에서 아직 다운로드하지 않은 2단원(lesson02)의 차시로 이동해 봅니다.
확인 항목

시나리오 5. 상태 유지

뷰어를 껐다 켜도 다운로드 상태가 유지되는지 확인합니다.

다시 다운로드하라고 하거나, 다운로드한 단원이 열리지 않으면 안 됩니다.

절차
  1. 뷰어를 완전히 종료합니다.
  2. Start.exe로 뷰어를 다시 실행합니다.
  3. 단원 선택에서 다운로드했던 단원을 선택해 보고, 콘텐츠도 열어 봅니다.
확인 항목

시나리오 6. 중단·오류 상황

다운로드가 중간에 실패하거나 끊겨도 문제가 없는지 확인합니다.

실패해도 뷰어가 멈추지 않고, 다시 시도하면 정상적으로 설치되어야 합니다.

절차 A — 네트워크 끊기
  1. 미다운로드 단원의 다운로드 버튼을 누릅니다.
  2. "다운로드 중..." 진행 중에 네트워크를 끊습니다. (랜선 분리 또는 Wi-Fi 끄기)
  3. 실패 알림을 확인한 뒤 네트워크를 다시 연결하고, 같은 단원을 다시 다운로드합니다.
확인 항목
절차 B — 진행 중 강제 종료
  1. 처음 상태로 되돌린 뒤(테스트 준비 참고), 단원 다운로드를 시작합니다.
  2. "다운로드 중..." 또는 "설치 중..." 진행 중에 뷰어를 강제 종료합니다. (창 닫기 또는 작업 관리자)
  3. 뷰어를 다시 실행하고 같은 단원을 다시 다운로드합니다.
