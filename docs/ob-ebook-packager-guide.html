소개

ebook ebook 패키저는 PDF를 불러와 인터랙티브 교과서(ebook)를 만드는 저작 도구입니다. 페이지 위에 버튼을 배치해 다른 페이지·외부 URL로 연결하고, 목차·출판 정보·검색 내용을 편집한 뒤 ZIP 파일로 저장합니다.

  • 입력: PDF 파일 (교과서 원본)
  • 출력: 교과서 ZIP — 페이지 HTML·이미지·버튼·목차가 포함된 패키지
  • 작업 저장: 저장한 ZIP을 다시 불러와 이어서 편집할 수 있습니다

사용 전 참고 사항

  • 실행 환경 — Windows 데스크톱 앱입니다. 별도 설치 과정 없이 배포된 실행 파일을 실행하면 됩니다.
  • 결과물 — 패키징 결과는 .zip 파일 1개입니다. 내용은 수정하지 말고 그대로 전달하거나 아래 위치에 배치하세요.
  • 뷰어 배치 — ZIP의 압축을 풀어 뷰어의 platform/onbook/ebook/(책 폴더)/ 바로 아래에 내용물(book.xml·toc.xml·web/)이 오도록 배치합니다. 책 폴더는 책(권) 단위입니다 — unit01: 책 1, unit02: 책 2(수학 익힘책처럼 2권으로 구성되는 경우).
    예: platform/onbook/ebook/unit01/book.xml (웹 배포판은 platform/ 없이 onbook/ebook/…)
  • 확인 방법 — 배치 후 OnBook 뷰어를 실행해 실제 표시를 확인합니다.

영역 설명
메뉴바 (상단) 프로젝트·편집·정렬·도움말 메뉴. 주요 기능은 단축키로도 실행할 수 있습니다.
퀵 툴바 자주 쓰는 기능(저장·실행 취소 등)을 빠르게 실행하는 아이콘 모음입니다.
페이지 목록 (왼쪽) 페이지 썸네일 목록. 드래그로 순서를 바꾸고, 페이지를 선택·삭제합니다.
편집 영역 (중앙) 선택한 페이지를 양면(펼침면)으로 표시합니다. 버튼 배치·선택·이동이 이곳에서 이루어집니다.
설정 패널 (오른쪽) 프로젝트 설정(시작 페이지)·목차 설정·출판 정보와, 선택한 페이지/버튼의 속성을 편집합니다.

기본 작업 흐름

  1. PDF 불러오기 — 메뉴 프로젝트 → PDF 페이지 추가 (Alt+D)로 PDF를 불러옵니다. 새로 시작할 때는 새 프로젝트 (Ctrl+N)로 초기화 후 진행합니다.
  2. 페이지 정리 — 왼쪽 목록에서 순서 변경·삭제하고, 필요하면 편집 → 빈 페이지 추가 (Ctrl+Insert)로 빈 페이지를 끼워 넣습니다.
  3. 버튼 만들기 — 버튼 타입과 디자인을 정의하고, 페이지 위에 드래그로 배치한 뒤 링크(페이지 이동·외부 URL)와 툴팁을 연결합니다.
  4. 검색 내용 확인 — PDF에서 자동 추출된 페이지별 검색 텍스트를 확인·수정합니다.
  5. 목차·출판 정보 입력 — 목차 항목(계층 구조)과 출판 정보(제목·저자·출판사 등), 시작 페이지 번호를 입력합니다.
  6. ZIP 저장 — 프로젝트 → ZIP 파일 저장 (Ctrl+S)으로 결과물을 저장합니다.

PDF 페이지 추가 (삽입·교체)

프로젝트 → PDF 페이지 추가에서 PDF를 열고, 원하는 페이지를 선택해 현재 프로젝트에 삽입하거나 기존 페이지를 교체합니다.

  • 순차 모드 — 선택한 페이지들을 지정 위치부터 순서대로 삽입/교체합니다.
  • 비순차 모드 — 페이지마다 위치를 지정해 삽입/교체합니다.
  • 검색 내용 유지 — 교체 시 기존 페이지의 검색 텍스트를 유지할지 선택합니다.
개정판 PDF로 일부 페이지만 바꾸는 경우, 전체를 다시 만들 필요 없이 교체 + 검색 내용 유지로 처리하면 기존 버튼·검색 편집을 보존할 수 있습니다.

버튼 만들기

버튼은 페이지 위에 올라가는 클릭 영역입니다. 뷰어에서 이 버튼을 누르면 지정한 페이지로 이동하거나 외부 URL이 열립니다.

1) 버튼 타입·디자인 정의

버튼 타입을 추가하고 타입별 디자인을 설정합니다. 같은 타입의 버튼은 디자인을 공유하므로, 용도별(예: 동영상·음성·활동지)로 타입을 나누는 것을 권장합니다.

항목설명
크기버튼의 너비·높이 (px)
배경색·투명도배경 색상과 투명도 (0~1)
테두리테두리 색·투명도·두께·둥글기
배경 이미지기본(Normal)·마우스 오버(Hover) 이미지 업로드

2) 배치·선택

  • 버튼을 페이지 위로 드래그해 배치하고, 드래그로 위치를 조정합니다.
  • 빈 곳에서 드래그하면 선택 영역이 만들어져 여러 버튼을 한 번에 선택할 수 있습니다.
  • 다중 선택 상태에서 복사·붙여넣기·삭제·이동·정렬을 일괄 수행합니다.
  • 모든 버튼 작업은 실행 취소(Ctrl+Z)·다시 실행(Ctrl+Y)이 가능합니다.

3) 링크·툴팁 연결

링크 종류설명
교과서 페이지 이동같은 교과서 안의 지정 페이지로 이동합니다.
외부 URL지정한 URL을 엽니다 (열기 대상 지정 가능).
툴팁버튼에 마우스를 올렸을 때 표시할 안내 문구입니다.

검색 내용 편집

PDF를 불러올 때 페이지별 텍스트가 자동 추출되어 뷰어의 검색에 사용됩니다. 추출 결과가 부정확하거나 검색에 노출하고 싶지 않은 내용이 있으면 검색 내용 편집에서 페이지별로 수정합니다.

목차 편집

뷰어의 목차 화면에 표시될 항목을 편집합니다. 항목 추가·수정·삭제와 함께 계층 구조(상위/하위 항목)와 연결 페이지 번호를 설정합니다. 드래그로 항목 순서와 계층을 조정할 수 있습니다.

목차를 입력하지 않고 저장하면 Page 1, Page 2, … 형태의 기본 목차가 자동 생성됩니다. 배포용 결과물에는 실제 목차를 입력하는 것을 권장합니다.

출판 정보·시작 페이지

항목설명
Book ID교과서 고유 ID (UUID). 새 프로젝트 생성 시 자동 발급됩니다.
제목교과서 제목. ZIP 저장 시 기본 파일명으로 사용됩니다.
저자 / 출판사 / 설명교과서 메타 정보입니다.
시작 페이지 번호첫 페이지에 부여할 실제 페이지 번호입니다. 표지·속지 때문에 본문이 1페이지부터 시작하지 않는 경우에 사용합니다 (프로젝트 설정에서 입력).

ZIP 저장·불러오기

  • 저장 (Ctrl+S) — 저장 위치와 파일명을 지정하면 페이지 이미지·썸네일·HTML·버튼·목차·출판 정보가 모두 포함된 ZIP이 생성됩니다. 기본 파일명은 출판 정보의 제목입니다.
  • 불러오기 (Ctrl+E) — 저장했던 ZIP을 열면 페이지·버튼·목차·출판 정보·시작 페이지 번호가 복원되어 이어서 편집할 수 있습니다.
ZIP 파일이 작업 파일이자 배포 파일입니다. 별도의 프로젝트 파일이 없으므로, 편집을 이어가려면 저장한 ZIP을 보관하세요.

출력 ZIP 구조

저장된 ZIP 내부는 다음과 같이 구성됩니다. 내용을 직접 수정하지 마세요.

(제목).zip
├─ book.xml                 ← 출판 정보·페이지 목록(spine)·시작 페이지 번호
├─ toc.xml                  ← 목차 (계층 구조)
└─ web/
   ├─ page-1.html …         ← 페이지 HTML
   ├─ css/common.css
   ├─ js/page-viewer.js
   ├─ js/ob/onbook.api.min.js
   ├─ page_images/page_001.jpg …      ← 페이지 원본 이미지
   ├─ page_thumbnails/page_001.jpg …  ← 페이지 썸네일
   ├─ page_data/page-1.js …           ← 페이지 데이터(버튼·검색 텍스트)
   └─ button_images/ …                ← 버튼 배경 이미지

book.xml

출판 정보(메타)와 페이지 순서(spine)를 담습니다.

<book xml:lang="ko-KR" version="1.0">
    <meta> … id·title·creator·publisher·description … </meta>
    <manifest><toc href="toc.xml"/></manifest>
    <spine startPageNumber="1">
        <page id="page-1" href="page-1.html" pageNumber="1"/>
        …
    </spine>
</book>

toc.xml

목차 편집에서 입력한 항목이 계층 그대로 기록됩니다.

<toc version="1.0">
    <nav>
        <item>
            <label>1단원. …</label>
            <content section="page-1" pageNumber="1" href="page-1.html"/>
            <item> … 하위 항목 … </item>
        </item>
    </nav>
</toc>

문제 해결

증상확인 사항
PDF가 열리지 않음 PDF가 암호화되어 있거나 손상되지 않았는지, 파일 경로에 특수 문자가 없는지 확인하세요.
ZIP 저장 실패 저장 경로에 쓰기 권한이 있는지, 디스크 공간이 충분한지 확인하세요.
버튼이 보이지 않음 페이지를 다시 선택해 보고, 버튼이 페이지 영역 밖에 배치되지 않았는지 확인하세요.
동작이 느림 페이지 수가 많으면 로딩·저장에 시간이 걸립니다. 버튼이 많은 페이지에서는 다중 선택 성능이 저하될 수 있습니다.
